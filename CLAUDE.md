# Инструкции Claude Code для Observer

## Обзор проекта
Observer — это **персональный** AI мета-агентный проект для одного пользователя.
Это НЕ multi-tenant приложение, НЕ SaaS, НЕ продукт для публичного использования.

Соответственно:
- Нет необходимости в сложной системе авторизации между пользователями
- API key достаточен для защиты endpoints
- Session/device ownership проверки не нужны (один пользователь = один владелец всего)

Строго следуйте этим рекомендациям.

## Стиль кода
- TypeScript: strict режим, никаких any
- Python: type hints везде, async/await
- Используйте осмысленные имена переменных
- Комментарии только для сложной логики

## Правила UI
- НИКАКИХ ЭМОДЗИ в интерфейсе
- Только иконки Lucide React
- Только тёмная тема
- Анимации через Framer Motion
- Адаптивный дизайн

## Git
- Конвенциональные коммиты (feat:, fix:, chore:)
- Одна фича на коммит
- Понятные сообщения коммитов

## Тестирование
- Пишите тесты для критичных путей
- Сервисы должны иметь unit-тесты

## Безопасность
- Никогда не коммитьте секреты
- Валидируйте все входные данные
- Санитизируйте пользовательские данные

## Версионирование
При каждом обновлении приложения менять версию в 4 местах:
1. `apps/desktop/src-tauri/tauri.conf.json` - версия бинарника
2. `apps/desktop/src-tauri/Cargo.toml` - версия Rust пакета
3. `apps/desktop/package.json` - для UI через __APP_VERSION__
4. `apps/web/package.json` - fallback версия на странице загрузки
5. Версии должны совпадать во всех четырёх файлах
6. Использовать semver: MAJOR.MINOR.PATCH
   - MAJOR: breaking changes
   - MINOR: новые фичи, редизайн UI
   - PATCH: багфиксы

После мержа в main:
- GitHub Actions автоматически создаст релиз
- Автообновление в приложении подтянет новую версию
